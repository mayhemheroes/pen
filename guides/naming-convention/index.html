Naming convention

KindCase styleExamples
VariablesCamel casefooBar, FooBar, i, x
FunctionsCamel casefooBar, FooBar, f, g
TypesCamel casefooBar, FooBar
ModulesUpper camel caseFooBar
Module directoriesUpper camel caseFooBar
PackagesKebab casegithub.com/foo-bar/baz-blah

Global and local names

  • Use descriptive names for top-level functions and types.
  • Use abbreviated names for function-local ones.
    • i for index
    • c for requestCount
    • sys for system
    • ctx for context

Acronyms

Acronyms are treated as single words.

  • Cpu
  • Ast
